Architecture Approaches

How to structure subscriptions, organizations, and the invitation system

A

Organization-Centric Model

New Organization entity owns subscriptions and buildings. The buyer creates an org during checkout. Buildings belong to an org. Plans are tied to orgs.

Pros

  • Clean separation: billing lives at org level, operations at building level
  • Natural fit for management companies with multiple buildings
  • Easy to add org-level features later (cross-building reports, consolidated billing)

Cons

  • Extra entity to manage — even single-building owners need an org
  • More complex data model and queries
  • Bigger migration — touches auth context, middleware, most API routes
B

Subscription-on-Building Model

No org entity. Each building has its own subscription. A "billing owner" user is the one who pays. Management companies are just users with SUPER_ADMIN across multiple buildings.

Pros

  • Simpler data model — extends existing Building with subscription fields
  • Smaller migration surface — no new auth context layer
  • Easier to reason about: one building = one subscription = one Stripe customer

Cons

  • Management companies managing 20 buildings need 20 separate subscriptions
  • No consolidated billing or cross-building view
  • Harder to add org-level features later (would need to retrofit the org layer)
C

Hybrid: Lightweight Account Layer

New Account entity (not a full org) that holds the Stripe subscription and owns buildings. Single-building buyers get an auto-created account. Management companies get a shared account. The account is mostly invisible in the UI for single-building users.

Pros

  • Supports both buyer types without over-engineering
  • Single-building users never see the account layer
  • Consolidated billing for management companies
  • Clean upgrade path to full org features if needed

Cons

  • Still an extra entity, but thinner than a full org
  • Need to handle auto-creation logic during checkout
  • "Invisible but present" layer can be confusing if not documented well

Recommendation: Option C (Hybrid)

It handles both buyer types cleanly, keeps the UX simple for individual managers, and gives management companies the consolidated experience they expect — without building a full organization management system upfront.
